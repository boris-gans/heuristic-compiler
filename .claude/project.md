# Project Guidance

## What this repo is for
Use this file for project-specific details that should not be mixed into the portable summary in `context.md`.

Examples:
- business purpose
- supported features
- API scope
- domain terminology
- non-goals
- team preferences specific to this repo

## Default assumptions
Until the repo says otherwise:
- this is a backend-oriented service
- external integrations are unreliable and should be treated defensively
- API behavior should be explicit and predictable
- maintainability matters more than clever optimization
